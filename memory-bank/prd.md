# 个人摄影网站需求与设计文档 (PRD)

## 1. 项目概述

- **项目定位**：个人高品质摄影作品展示平台，建立专属的数字名片。
- **核心设备端**：优先适配桌面端 (PC) 网页沉浸式浏览，兼顾移动端基础展示自适应。
- **视觉语言**：极简设计 (Minimalism)，全局深色模式 (Dark Mode) 为主色调，剥离冗余 UI，通过高对比度突出影像本身的色彩与光影。

## 2. 核心功能模块

### 2.1 前端展示侧 (C端)

- **全局导航**：极简顶部或侧边导航栏，包含：首页、风光、肖像、器材、关于。

- **首页 (Home)**
  - 大画幅轮播图或精选作品集瀑布流，作为视觉锚点定调。

- **作品列表页 (Gallery)**
  - 支持按“风光”、“肖像”等标签进行分类筛选与展示。
  - 性能优化：列表统一展示压缩后的缩略图，配合懒加载 (Lazy Load) 机制和占位符，确保页面秒开。

- **作品详情页 (Lightbox)**
  - 交互：点击缩略图弹出沉浸式暗框大图，触发加载高清原图。
  - EXIF 数据：在大图下方或侧边，使用极简无衬线字体展示核心拍摄参数。
    - 示例：`Nikon D800 | 35mm | f/1.4 | 1/200s | ISO 100`
  - 图文排版：支持动态渲染，若后台配置了创作手记则显示，无描述则自动隐藏并留白。
  - 互动专区：底部提供“点赞 (Like)”按钮及轻量级的评论留言模块。

- **器材与关于页 (Gear & About)**
  - 个人简介与摄影理念阐述。
  - 专属器材清单模块（结构化展示主力机身、镜头群及后期工具）。

### 2.2 后台管理侧 (CMS Admin)

- **身份验证**：安全的管理员专属登录入口及鉴权机制。

- **内容发布与管理**
  - 图床管理：支持图片拖拽上传。
  - EXIF 识别：上传时自动读取图片的 EXIF 信息，并提供表单供手动修正/补全。
  - 属性设置：下拉选择作品分类标签（风光/肖像）。
  - 内容编辑：可选填文字描述或拍摄手记。

- **互动数据审核**
  - 提供点赞数据统计概览。
  - 提供评论留言的管理列表，支持查看、隐藏和删除不友好的留言。

## 3. 技术架构设计

- **前端框架**：Next.js (React)
  - 支持良好的首屏渲染和路由管理，极度契合 AI 辅助编程生成。

- **UI 与样式引擎**：Tailwind CSS
  - 提供原子化 CSS，便于快速全局切换和控制暗黑模式细节。

- **后端、数据库与图床 (BaaS)**：Supabase
  - Supabase Storage：作为对象存储，用于存放高清原片，并利用其图片转换功能输出前端所需的压缩缩略图。
  - Supabase Database (PostgreSQL)：用于结构化存储照片元数据 (EXIF)、分类关系映射、点赞计数器以及评论内容。
