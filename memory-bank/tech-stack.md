# 摄影网站技术栈选型文档 (Tech Stack)

## 核心选型理念：AI 友好 + 极简全栈 + 影像级性能

对于 `wyx_photo_gallery` 项目，我们追求的是“一个人就是一支团队”的开发体验。本选型摒弃了复杂的前后端分离和自建服务器运维，采用目前业界最成熟的 **BaaS (后端即服务) + 现代前端框架** 模式。这套方案对于 AI 编程工具（如 Cursor, Bolt.new）来说理解率极高，生成的代码几乎可以直接运行。

### 1. 前端架构 (Frontend)
* **核心框架**：`Next.js` (App Router 模式)
  * **稳健理由**：目前 React 生态的绝对霸主，AI 工具极其擅长写 Next.js 代码。
  * **摄影特供优势**：内置 `<Image>` 组件，自动处理图片的懒加载、格式转换（WebP）和多尺寸适配，完美解决 D800 高清原片导致首页加载缓慢的问题。
* **样式引擎**：`Tailwind CSS`
  * **稳健理由**：无需手写复杂的 CSS 文件，直接在代码里写原子类（如 `bg-black text-white`）。极其方便实现全站“深色模式 (Dark Mode)”的统一控制。
* **UI 组件库**：`shadcn/ui` + `Lucide Icons`
  * **稳健理由**：它不是一个臃肿的第三方库，而是直接把极简风格的代码复制进你的项目中，你可以拥有 100% 的修改权，非常契合你所需要的极简画廊风格。

### 2. 后端、数据库与图床 (Backend & Storage)
* **一站式平台**：`Supabase`
  * **稳健理由**：这是开源界最强健的 Firebase 替代品。你不需要自己去配置云服务器、写后端的增删改查接口，它为你打包了一切。
  * **具体应用**：
    * **Database (PostgreSQL)**：极其强大稳健的关系型数据库，用来存照片的 EXIF 信息、分类标签、点赞数和评论。
    * **Storage (对象存储)**：内置的图床，用来存放你几十兆的高清原片。
    * **Auth (身份验证)**：极其简单的邮箱/密码登录机制，为你保护后台上传入口。

### 3. 部署方案 (Deployment)
* **前端托管**：`Vercel`
  * **稳健理由**：Next.js 的官方母公司，支持“Git 推送即部署”。你只需要把代码推送到 GitHub，Vercel 会自动给你分配免费的 HTTPS 域名并上线网站，零配置，速度极快。

### 4. 为什么这是“最简单但最稳健”的组合？
1. **防碎性强**：Next.js 负责界面的展示和交互，Supabase 负责存图和存数据，两者的边界极其清晰。
2. **零服务器运维**：不用去买阿里云 ECS，不用配置 Nginx，不用折腾宝塔面板。完全 Serverless 化。
3. **AI 成品率最高**：目前所有主流 AI 编程工具，用 `Next.js + Tailwind + Supabase` 生成全栈项目的成功率和代码规范度是最高的。
